🚨 SRE Incident Response Workflow

A practical command-driven checklist for a production incident.

1. Triage

  • Check service health: systemctl status app
  • Check load: uptime, top
  • Check recent errors: journalctl -u app -n 200
Step 1

2. Isolate

  • Check network/ports: ss -tulnp
  • Check disk space: df -h
  • Check memory: free -h
Step 2

3. Mitigate

  • Restart the affected service: systemctl restart app
  • Rollout restart in Kubernetes: kubectl rollout restart deployment/app
  • Scale up if resource-bound: kubectl scale deployment/app --replicas=5
Step 3

4. Root Cause

  • Stream live logs: tail -f app.log
  • Grep for the error pattern: grep -n "Exception" app.log
  • Correlate with a recent deploy: git log --oneline -5
  • Check metrics: PromQL rate(http_requests_total{status="5xx"}[5m])
Step 4

5. Document

  • Write a postmortem: timeline, impact, root cause, action items.
  • Add monitoring/alerts to catch it earlier next time.
  • Share learnings with the team.
Step 5
